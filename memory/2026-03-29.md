### Critical Insight
**The "memory loss" isn't a bug — it's architectural:**
- Felix sessions are intentionally stateless
- Persistence requires explicit file documentation
- Procedural knowledge must be encoded in:
  - SKILL.md files (for complex workflows)
  - Protocol docs (like NYLONGERIE_POSTING_PROTOCOL.md)
  - Reference in AGENTS.md (so it's always loaded)
  - Scripts with clear usage comments

**Solution Pattern:**
1. Document complete workflows in markdown
2. Reference them in AGENTS.md or via skills
3. Automate via scripts (not manual steps)
4. State tracking via JSON files (queue.json, operations.json, etc.)
